# Index Portal Guide

## 项目简介

本项目是一个用于归档和发布多个独立 HTML 页面的仓库。仓库本身不针对任何特定域名或具体网站，仅作为静态页面的索引与分发入口。

通过此仓库，您可以浏览、获取或引用其中收录的各类独立 HTML 资源。每个页面均为自包含的静态文件，便于离线查看或进一步集成。

## 目录说明

仓库根目录下包含以下内容：

- `index.html` — 仓库首页，可作为所有归档页面的导航入口
- `pages/` — 存放独立 HTML 页面的子目录，按功能或用途分类
  - 每个子文件夹内包含一个或多个 `.html` 文件
  - 文件夹命名尽量反映页面类型或主题
- `assets/` — 页面共用的样式、脚本或图片资源（如有）
- `README.md` — 本文件

## 页面归档说明

所有归档页面均遵循以下原则：

- 每个页面均为完整的独立 HTML 文件，不依赖外部网络资源（除非特别说明）
- 页面内容保持原始结构，不添加额外追踪或跳转逻辑
- 归档文件命名清晰，便于快速识别用途
- 不包含任何与特定商业、推广或诱导点击相关的元素

如需新增归档页面，请参考“维护说明”中的操作指引。

## 维护说明

### 新增页面

1. 在 `pages/` 下创建合适的子目录（如已有分类，请放入对应目录）
2. 将独立 HTML 文件放入该目录
3. 更新 `index.html` 中的导航链接（如有）
4. 提交 Pull Request 或直接推送至主分支（视仓库权限而定）

### 更新页面

- 直接替换对应文件即可，请保持文件名不变
- 如涉及重大结构变更，请在提交信息中注明

### 删除页面

- 移除对应文件及目录
- 同步更新 `index.html` 中的相关链接

### 注意事项

- 请勿上传包含敏感信息、恶意代码或违反法律法规的内容
- 保持目录结构简洁，避免冗余嵌套
- 鼓励为每个页面添加简单的内联注释（如文件头部说明）
